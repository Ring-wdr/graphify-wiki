# Git 체리픽과 리버트

## 한 줄 요약

체리픽은 필요한 변경만 다른 브랜치로 옮길 때 쓰고, 리버트는 이미 반영된 변경을 새 커밋으로 되돌릴 때 쓴다.

## 언제 쓰나

- 핫픽스 커밋 하나만 릴리스 브랜치로 옮기고 싶을 때
- 잘못 배포된 변경을 히스토리를 보존한 채 되돌리고 싶을 때
- 특정 커밋의 영향만 분리해서 검증하고 싶을 때

## 핵심 개념

- 체리픽은 특정 커밋의 diff를 현재 브랜치에 재적용한다.
- 리버트는 기존 커밋을 삭제하지 않고 반대 변경을 새 커밋으로 만든다.
- 여러 파일이 섞인 큰 커밋은 체리픽과 리버트 둘 다 다루기 어려워진다.

## 실전 예시

```bash
git cherry-pick <commit-sha>
git revert <commit-sha>
```

## 주의할 점

- 체리픽 대상 커밋이 다른 선행 변경에 의존하면 충돌이나 누락이 생길 수 있다.
- 리버트는 "되돌림 자체도 의미 있는 변경"이므로 배포 메모와 연결해 이해해야 한다.
- 커밋 단위가 너무 크면 원하는 변경만 안전하게 옮기기 어렵다.
