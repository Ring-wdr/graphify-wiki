# Git 충돌 해결 체크리스트

## 한 줄 요약

충돌 해결은 텍스트를 합치는 작업이 아니라 두 변경의 의도를 재구성하는 작업이다.

## 언제 쓰나

- 리베이스나 머지 도중 충돌이 발생했을 때
- 코드와 설정이 서로 다른 방향으로 바뀐 브랜치를 합칠 때
- 급한 핫픽스 이후 장기 브랜치와 다시 합쳐야 할 때

## 핵심 개념

- 충돌 구간 앞뒤 문맥을 함께 읽어야 실제 의도 차이를 알 수 있다.
- 정답이 둘 중 하나인 경우도 있지만, 두 변경을 다시 조합해야 하는 경우가 더 많다.
- 해결 후에는 "컴파일 또는 테스트 통과"보다 "변경 의도 유지"를 먼저 점검해야 한다.

## 실전 예시

```bash
git status
git diff
git add <resolved-files>
git rebase --continue
```

## 주의할 점

- 기계적으로 ours 또는 theirs를 고르면 숨은 회귀가 남기 쉽다.
- 충돌 해결 커밋에 다른 리팩터링을 섞으면 리뷰가 어려워진다.
- 충돌 원인이 큰 커밋이라면 이후에는 커밋 단위를 더 줄이는 쪽이 낫다.
