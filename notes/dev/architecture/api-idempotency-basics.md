# API 멱등성 기초

## 한 줄 요약

멱등성은 같은 요청이 여러 번 들어와도 시스템 결과가 안전하게 유지되도록 만드는 계약이다.

## 언제 쓰나

- 재시도 가능한 API를 설계할 때
- 결제, 주문, 작업 시작 같은 중복 실행 위험이 큰 작업을 다룰 때
- 네트워크 타임아웃 이후 클라이언트가 동일 요청을 다시 보낼 수 있을 때

## 핵심 개념

- 멱등성은 "중복 요청 자체"가 아니라 "중복 요청 이후의 결과 상태"를 다룬다.
- 클라이언트 키, 요청 키, 자연 키 중 무엇으로 중복을 판별할지 정해야 한다.
- 저장소와 메시지 처리 흐름이 멱등성을 함께 지켜야 진짜로 안전하다.

## 실전 예시

- `Idempotency-Key`를 받아 동일 키 요청에 같은 결과를 돌려준다.
- 작업 시작 API는 새 작업 생성 전에 기존 실행 이력을 확인한다.
- 재시도 가능한 소비자는 처리 결과를 기록한 뒤 중복 메시지를 건너뛴다.

## 주의할 점

- 응답 캐시만 같고 실제 내부 상태가 달라지면 멱등성이 아니다.
- 만료 정책이 짧으면 느린 재시도에서 중복 실행이 생길 수 있다.
- 외부 결제나 서드파티 호출은 내부 멱등성만으로는 보호되지 않는다.
